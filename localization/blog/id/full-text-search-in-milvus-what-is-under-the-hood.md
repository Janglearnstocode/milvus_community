    </button></h2><p>Meskipun pengambilan sematan padat unggul dalam pencarian semantik, algoritme pencarian teks lengkap seperti BM25 terus memainkan peran penting dalam tugas-tugas pengambilan, melengkapi pendekatan sematan padat dengan prediktabilitas dan kemampuannya untuk menangani terminologi khusus.</p>
<p>Sejak <a href="https://milvus.io/blog/introduce-milvus-2-5-full-text-search-powerful-metadata-filtering-and-more.md">versi 2.5</a>, Milvus memperkenalkan dukungan bawaan untuk pencarian teks lengkap dengan pendekatan Sparse-BM25, yang memungkinkan pengguna untuk dengan mudah mengintegrasikan kemampuan ini ke dalam jalur pencarian mereka tanpa memerlukan mesin pencari terpisah. Dalam blog ini, kami akan mengungkap cara kerja Sparse-BM25 di Milvus.</p>

<p>Kedua, pengguna tidak perlu lagi menghitung vektor sparse terlebih dahulu. Pengguna hanya perlu menyediakan dokumen dalam bentuk teks mentah selama proses pencarian. Milvus bertanggung jawab untuk memproses teks menjadi token dengan penghilangan stop-word dan stemming, dan memelihara statistik distribusi istilah global di seluruh korpus yang diperlukan untuk menghitung <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">IDFIDF</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.13889em;">IDF</span></span></span></span> dan <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">avgdlavgdl</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.01968em;">avgdl</span></span></span></span>.</p>
<p>Pada waktu penyisipan, teks diberi token dan diubah menjadi vektor jarang oleh Milvus dan disimpan dalam bidang vektor jarang. Pada waktu pencarian, Milvus mengkodekan kueri sebagai vektor <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">IDFIDF</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.13889em;">IDF</span></span></span></span> berdasarkan statistik global dan mengirimkannya ke QueryNode bersama dengan <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">avgdlavgdl</annotation></semantics></math></span></span> realtime <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.01968em;">avgdl</span></span></span></span> untuk melakukan pencarian Sparse-BM25.</p>

<h3 id="Step-1-Define-a-Collection-Schema" class="common-anchor-header">Langkah 1: Tentukan Skema Koleksi</h3><p>Pertama, kita perlu membuat koleksi dan mendefinisikan bidang untuk <strong>data teks mentah dan vektor jarang dokumen yang disandikan</strong> (kita hanya perlu memasukkan data teks mentah, vektor jarang dihitung oleh Milvus secara internal).</p>
<p>Pertama, kita mendefinisikan skema koleksi dengan bidang untuk teks mentah (dokumen) dan vektor jarang (sparse). Perhatikan bahwa kita juga mendefinisikan <code translate="no">Function</code> di dalam skema, yang memberi tahu Milvus untuk secara otomatis mengkonversi teks mentah yang dicerna menjadi vektor jarang yang digunakan untuk penilaian BM25.</p>
<pre><code translate="no" class="language-Python"><span class="hljs-keyword">from</span> pymilvus <span class="hljs-keyword">import</span> MilvusClient, DataType, Function, FunctionType

client = MilvusClient(uri=<span class="hljs-string">&quot;http://localhost:19530&quot;</span>)

<span class="hljs-comment"># Define a schema</span>
schema = client.create_schema()
schema.add_field(field_name=<span class="hljs-string">&quot;id&quot;</span>, datatype=DataType.INT64, is_primary=<span class="hljs-literal">True</span>, auto_id=<span class="hljs-literal">True</span>)
schema.add_field(field_name=<span class="hljs-string">&quot;document&quot;</span>, datatype=DataType.VARCHAR, max_length=<span class="hljs-number">1000</span>, enable_analyzer=<span class="hljs-literal">True</span>)
schema.add_field(field_name=<span class="hljs-string">&quot;sparse&quot;</span>, datatype=DataType.SPARSE_FLOAT_VECTOR)

<span class="hljs-comment"># Add BM25 function mapping text to sparse vector</span>
bm25_function = Function(
    name=<span class="hljs-string">&quot;text_bm25_emb&quot;</span>,
    input_field_names=[<span class="hljs-string">&quot;document&quot;</span>],
    output_field_names=[<span class="hljs-string">&quot;sparse&quot;</span>],
    function_type=FunctionType.BM25,
)
schema.add_function(bm25_function)
<button class="copy-code-btn"></button></code></pre>
<blockquote>
<p>💡 Pastikan enable_analyzer = True diatur agar bidang teks dapat dicari menggunakan BM25. Anda juga dapat mengonfigurasi penganalisis khusus jika diperlukan-lihat <a href="https://milvus.io/docs/analyzers.md">dokumentasi Penganalisis</a> untuk detailnya.</p>
