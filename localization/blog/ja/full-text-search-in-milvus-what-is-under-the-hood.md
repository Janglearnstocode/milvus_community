<p>第二に、ユーザーはスパースベクトルを事前に計算する必要がなくなった。ユーザーは、取り込み時や検索時に文書を生テキストで提供するだけでよい。Milvusはテキストをストップワード除去やステミングを施したトークンに処理し、<span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">IDFIDF</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.13889em;">IDFと</span></span></span></span> <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">avgdlavgdl</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.01968em;">avgdlを</span></span></span></span>計算するために必要なコーパス全体の用語分布統計を保持する。</p>
<p>挿入時にテキストはトークン化され、Milvusによってスパースベクトルに変換され、スパースベクトルフィールドに格納される。検索時には、milvusはグローバル統計情報に基づいてクエリを<span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">IDFIDF</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.13889em;">IDF</span></span></span></span>ベクトルとしてエンコードし、リアルタイム<span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><annotation encoding="application/x-tex">avgdlavgdl</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.01968em;">avgdlとともに</span></span></span></span>QueryNodeに送り、スパース-BM25検索を実行する。</p>

<h3 id="Step-1-Define-a-Collection-Schema" class="common-anchor-header">ステップ1: コレクションスキーマの定義</h3><p>まず、コレクションを作成し、<strong>生のテキストデータとエンコードされたドキュメントのスパースベクトルの</strong>フィールドを定義する必要があります（生のテキストデータを取り込むだけで、スパースベクトルはMilvus内部で計算されます）。</p>
<p>まず、生テキスト（document）とスパースベクトル（sparse）のフィールドを持つコレクションスキーマを定義する。スキーマには<code translate="no">Function</code> も定義し、Milvus に取り込まれた生テキストを BM25 スコアリングに使用するスパースベクトルに自動的に変換するように指示する。</p>
<pre><code translate="no" class="language-Python"><span class="hljs-keyword">from</span> pymilvus <span class="hljs-keyword">import</span> MilvusClient, DataType, Function, FunctionType

client = MilvusClient(uri=<span class="hljs-string">&quot;http://localhost:19530&quot;</span>)

<span class="hljs-comment"># Define a schema</span>
schema = client.create_schema()
schema.add_field(field_name=<span class="hljs-string">&quot;id&quot;</span>, datatype=DataType.INT64, is_primary=<span class="hljs-literal">True</span>, auto_id=<span class="hljs-literal">True</span>)
schema.add_field(field_name=<span class="hljs-string">&quot;document&quot;</span>, datatype=DataType.VARCHAR, max_length=<span class="hljs-number">1000</span>, enable_analyzer=<span class="hljs-literal">True</span>)
schema.add_field(field_name=<span class="hljs-string">&quot;sparse&quot;</span>, datatype=DataType.SPARSE_FLOAT_VECTOR)

<span class="hljs-comment"># Add BM25 function mapping text to sparse vector</span>
bm25_function = Function(
    name=<span class="hljs-string">&quot;text_bm25_emb&quot;</span>,
    input_field_names=[<span class="hljs-string">&quot;document&quot;</span>],
    output_field_names=[<span class="hljs-string">&quot;sparse&quot;</span>],
    function_type=FunctionType.BM25,
)
schema.add_function(bm25_function)
